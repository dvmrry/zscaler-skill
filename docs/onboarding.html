Zscaler · technical overview 01 / 18
01 — In plain English 02 / 18

Zscaler is the cloud that sits between your users and everything they want to reach — the internet, SaaS apps, and your own private apps. Every connection goes through it first.

Everything else on the next 16 slides is a detail of how that happens, which traffic, and what gets inspected along the way.

02 — Why it exists 03 / 18

It collapses three categories
of legacy infrastructure.

ReplacesWeb gateways & firewalls
Outbound user-to-internet inspection moves from on-prem appliances to a globally distributed cloud edge.
ReplacesVPN concentrators
Users authenticate per-app instead of per-network. No more “on the VPN, on the LAN.”
ReplacesInline inspection appliances
DLP, sandbox, IPS, malware scanners, isolation — consolidated into the same cloud pipeline.

Part 3 · The single most important thing 14 / 18
Memorize this asymmetry

ZIA defaults to allow.
ZPA defaults to block.

Source — Zscaler help docs, internal policy-evaluation reference

CONSEQUENCE
Web traffic with no rule match → user reaches the site.
Private app with no rule match → user is denied.
"Why can't anyone reach the new internal tool?"
— usually means no ZPA rule was added.
"Why is malicious-looking-domain.xyz reachable?"
— usually means it didn't match a block rule.

Web module pipeline.

Within the ZIA web module, policies evaluate in order. The first violation halts the pass — rules below never run. Don't internalize the full list today; internalize that the order matters and CAC sits before URL Filtering.

P2P Control

+ HTTPS two-pass: SNI/CONNECT pass uses domain-only (Malicious URLs → CAC → URL Filter → Bandwidth). SSL Inspection then decides Inspect / Bypass. If decrypted, the full pipeline runs again with URL path + headers + body visible. POST adds DLP, Malware, File Type. Response adds Sandbox.

Part 2 · Specialized layers 16 / 18

Specialized products you'll
encounter, but won't run day one.

ZDX EXPERIENCE

User-experience monitoring. Probes, ZDX score, deeptraces. The "is the user's day actually working?" product.

ZBI ISOLATION

Cloud Browser Isolation. Risky URLs render in Zscaler's cloud; pixels stream to the browser. Triggered by URL Filter or Smart Browser Isolation.

ZWA DLP OPS

Workflow Automation. Manages ZIA-DLP incidents. Auto-close, auto-notify, auto-escalate; ServiceNow / Slack hooks.

AppProtection WAF/IPS

Inline WAF for ZPA. OWASP CRS, ThreatLabZ, AD-protocol controls. Lives inside ZPA.

CBC / ZTW CLOUD & BRANCH

VM-based forwarding for cloud workloads (AWS / Azure / GCP) and branch offices. ZCC's equivalent for non-user traffic.

Risk360 QUANTIFICATION

Cyber-risk dashboard. Monte Carlo financial-loss simulation. Surfaces cyber exposure in financial terms — often referenced in board and executive reviews.

Deception ACTIVE DEFENSE

Decoys catch lateral movement. Any interaction = breach signal. Integrates with ZPA via Zero Trust Network decoys.

AI Guard LLM SECURITY

Prompt-injection, jailbreak, sensitive-data detection on LLM traffic. 15 detector categories. Layers on top of ZIA GenAI URL Filter.

Part 4 · Action 17 / 18
Onboarding checklist

What to do this week.

Not theoretical. Concrete actions to make the platform real before you're in the deep end on a real incident.

01
Get a tenant login & reset to read-only
Through ZIdentity. Read-only RBAC scope. You can break less than you think with read-only, but no need to test that today.
02
Walk one URL through ZIA in the admin console
Pick any domain. Look at: which categories cover it, which URL Filter rule matches, what action fires, whether SSL inspects.
03
Walk one app through ZPA
Pick an internal app. Look at: app segment, segment group, server group, which connector group serves it, which access policies reference it.
04
Read the primer + portfolio-map references
references/_meta/primer/ and references/_meta/portfolio-map.md. ~45 minutes total.
05
Find a colleague with battle scars — ask what bit them last quarter
Tribal knowledge. The reference docs cover the mechanics; the war stories cover what actually goes wrong.
06
Ask one stupid question in #zscaler this week
It won't be stupid. The asymmetries (default-allow vs default-block, CAC-vs-URL-filter) bite everyone.
